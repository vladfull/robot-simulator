# Robot API Reference

Кожен ваш алгоритм — це функція `control_step(robot)`, яку симулятор
викликає на кожному кроці симуляції (за замовчуванням 50 разів на
секунду). Параметр `robot` — це екземпляр `RobotAPI`: безпечний
фасад навколо реального об'єкта робота. Він єдиний легальний шлях
для зчитування стану і подачі команд.

> Тільки `set_velocity(...)` змінює стан робота. Усі інші методи —
> read-only.

---

## Швидкий старт

```python
def control_step(robot):
    if robot.distance_to_goal() < 0.2:
        robot.set_velocity(0.0, 0.0)
        return
    error = robot.angle_to_goal() - robot.get_orientation()
    robot.set_velocity(0.5, 2.0 * error)
```

Цей мінімальний приклад: пропорційний регулятор по куту до цілі плюс
постійна лінійна швидкість.

---

## Read-only методи

### `get_position() -> (x, y)`

Поточні координати робота у метрах. `x` — горизонталь (вправо +),
`y` — вертикаль (вгору +).

```python
x, y = robot.get_position()
```

### `get_orientation() -> float`

Курс робота у радіанах, нормалізований у проміжок `[-pi, pi]`. `0`
означає «дивиться у бік +X», `pi/2` — «у бік +Y».

```python
theta = robot.get_orientation()
```

### `get_velocity() -> (linear, angular)`

Поточна лінійна (м/с) і кутова (рад/с) швидкості — те, що насправді
застосовано до робота після обмежень.

### `get_sensor_data() -> list[float]`

Список з 16 відстаней від робота до найближчої перешкоди вздовж
кожного променя. Промені рівномірно розподілені на 360°, починаючи з
напрямку «вперед» (індекс 0) і йдуть проти годинникової стрілки.

```python
distances = robot.get_sensor_data()
front = distances[0]
left = distances[len(distances) // 4]
back = distances[len(distances) // 2]
right = distances[3 * len(distances) // 4]
```

Якщо промінь не зустрів перешкоди в межах `get_sensor_max_range()`,
повертається саме це максимальне значення.

### `get_sensor_count() -> int`

Кількість променів сенсора (зараз 16, але краще не закладатися на це
число).

### `get_sensor_max_range() -> float`

Максимальна дальність сенсора у метрах (зараз 5.0).

### `get_goal_position() -> (gx, gy)`

Координати цільової точки.

### `distance_to_goal() -> float`

Евклідова відстань від робота до цілі.

### `angle_to_goal() -> float`

Курс на ціль у радіанах. Зручно віднімати від `get_orientation()`,
щоб отримати помилку курсу:

```python
heading_error = robot.angle_to_goal() - robot.get_orientation()
# Не забудьте нормалізувати у [-pi, pi]:
while heading_error > math.pi:
    heading_error -= 2 * math.pi
while heading_error < -math.pi:
    heading_error += 2 * math.pi
```

### `get_time() -> float`

Симульований час у секундах від моменту старту/скидання симуляції.
Корисно для PID-регуляторів, таймерів станів тощо.

### `is_collision() -> bool`

Чи робот зараз торкається перешкоди.

### `get_world_size() -> (width, height)`

Розміри сцени у метрах.

## Команди

### `set_velocity(linear, angular)`

Подати керуючі сигнали:

* `linear` — лінійна швидкість у м/с (від'ємні значення = рух назад);
* `angular` — кутова швидкість у рад/с (додатні = поворот вліво / проти годинникової стрілки).

Значення поза межами робота автоматично обрізаються (за замовчуванням
±1.0 м/с і ±2.0 рад/с). NaN/Inf замінюються на 0. Виклик з нечисловими
значеннями просто переводить швидкість у 0 — поганий ввід нічого не
ламає.

```python
robot.set_velocity(0.5, 0.0)    # пряма вперед
robot.set_velocity(-0.3, 0.0)   # назад
robot.set_velocity(0.0, 1.5)    # на місці вліво
robot.set_velocity(0.5, -1.0)   # вперед і вправо
```

> Якщо `set_velocity` не викликати, робот зберігає попередню швидкість.
> Тобто алгоритм, який нічого не робить, не зупиняє робот — це часта
> джерело несподіваних результатів. Викликайте `set_velocity(0, 0)`
> явно, коли хочете зупинитися.

### `log(message)`

Записати рядок у консоль інтерфейсу. До нього автоматично додається
позначка симульованого часу.

```python
robot.log(f"Distance to goal: {robot.distance_to_goal():.2f}")
```

## Що дозволено в коді алгоритму

Доступне:

* модулі `math`, `random` (вже імпортовані глобально, без `import`);
* стандартні функції: `abs`, `min`, `max`, `len`, `range`, `round`,
  `sum`, `sorted`, `enumerate`, `zip`, `map`, `filter`, `list`,
  `dict`, `tuple`, `set`, `int`, `float`, `str`, `bool`, `print`,
  `isinstance`, `hasattr`, `getattr`, `repr`, `type`.

Заборонене:

* `import` будь-чого (уже імпортовано все потрібне);
* файлові операції (`open`, читання/запис файлів);
* `os`, `sys`, `subprocess`, `socket`, `__import__`, `eval`, `exec`,
  `compile`;
* запуск окремих потоків, мережа.

Спроба викликати заборонене зупинить ваш код і виведе помилку у
консоль; основний застосунок продовжить працювати.

## Збереження стану між кроками

`control_step` викликається багато разів. Глобальний словник на
рівні модуля — правильний спосіб зберегти інтегральну складову PID
або поточний режим:

```python
_state = {"integral": 0.0, "last_error": 0.0, "mode": "GO_TO_GOAL"}

def control_step(robot):
    error = robot.angle_to_goal() - robot.get_orientation()
    _state["integral"] += error * 0.02
    ...
```

Кожне натискання **Run** перезавантажує код, так що ваш `_state`
скидається до значень за замовчуванням — це бажана поведінка.

---

## Лайфсайкл одного кроку

1. Сенсори робота оновлюються по поточній позі.
2. Симулятор викликає ваш `control_step(api)`.
3. Ваш код читає API і викликає `api.set_velocity(v, w)`.
4. Кінематика інтегрується, рух перевіряється на колізії з
   axis-separated sliding (упирається у стіни і ковзає вздовж них).
5. Лог поточного стану записується.
6. Стан передається у візуалізацію.

Якщо ваш код кидає виняток, traceback потрапить у консоль, симуляція
зупиниться, але ви зможете виправити код і знову натиснути **Run**.
